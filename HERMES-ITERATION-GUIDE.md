### 1.5 How Pages Are Rendered

1. Every `SeoPage` object in `pages[]` (site.ts) or `aiToolPages[]` (ai-tools-data.ts) is merged by `getAllPageEntries()`.
2. `src/pages/pages/[slug].astro` calls `getStaticPaths()` → returns one route per page.
3. At build time, Astro generates `/pages/{slug}/index.html` for each entry.
4. The `[slug].astro` template renders sections by switching on `section.type`.
5. Schema (Article + FAQ) is injected as `<script type="application/ld+json">` in `<head>`.

## Chapter 2: Content Source 1 — Google Hot Keywords Pipeline

### Overview

This is the original core revenue pipeline. The flow: discover high-traffic search terms → build targeted SEO pages → rank in Google → monetize via ads and affiliate links.

### Step 1: Discover Hot Keywords

```bash
npm run google-trends
```

This runs `scripts/fetch-google-trends.ts` which outputs keyword data to `scripts/data/`.

**Expected output format** (JSON):
```json
[
  {
    "keyword": "best ai meeting assistant 2026",
    "searchVolume": 12000,
    "cpc": 4.50,
    "competition": "medium",
    "trend": "rising",
    "relatedQueries": ["ai meeting notes", "otter ai vs fireflies"]
  }
]
```

### Step 2: Analyze Keywords — Scoring Matrix

Evaluate each keyword using this quantitative framework:

| Factor | Weight | Scoring Criteria |
|--------|--------|-----------------|
| Monthly Search Volume | 30% | <500 = 1pt, 500-2K = 2pt, 2K-10K = 3pt, 10K-50K = 4pt, >50K = 5pt |
| CPC (Cost Per Click) | 25% | <$1 = 1pt, $1-3 = 2pt, $3-8 = 3pt, $8-15 = 4pt, >$15 = 5pt |
| Competition Level | 20% | Very High = 1pt, High = 2pt, Medium = 3pt, Low = 4pt, Very Low = 5pt |
| Trend Direction | 15% | Declining = 1pt, Stable = 2pt, Rising = 4pt, Breakout = 5pt |
| Content Fit | 10% | Poor = 1pt, Moderate = 3pt, Perfect = 5pt |

**Weighted Score Formula**:
```
Score = (Volume × 0.30) + (CPC × 0.25) + (Competition × 0.20) + (Trend × 0.15) + (Fit × 0.10)
```

### Step 3: Decision — Build or Skip

| Weighted Score | Decision |
|---------------|----------|
| ≥ 3.5 | **BUILD** — High priority, create page immediately |
| 2.5 – 3.4 | **CONSIDER** — Build if you have capacity, queue for next sprint |
| < 2.5 | **SKIP** — Not worth the effort at this stage |

Additional hard rules:
- **Always build** if CPC > $8 and search volume > 1000 (high commercial intent).
- **Always skip** if search volume < 200 (not enough traffic potential).
- **Always build** if trend = "breakout" regardless of other scores.

### Step 4: Generate Page Skeleton

```bash
npm run gen-page
```

This runs `scripts/generate-page-template.ts`. It outputs a partial `SeoPage` object you must complete.

Alternatively, manually create the SeoPage object directly in `src/lib/site.ts` by adding to the `pages[]` array:

```typescript

  sections: [],        // Fill in Step 5

```

### Step 5: Fill Content with seo-content-writer Skill

Invoke the `seo-content-writer` skill with this prompt template:

```
/seo-content-writer

Topic: [TARGET KEYWORD]
Target keyword: [PRIMARY KEYWORD]
Secondary keywords: [2-3 RELATED KEYWORDS]
Content type: [ARTICLE TYPE — guide/comparison/listicle/tutorial]
Word count: 1500-3000
Audience: [WHO — marketers/developers/small business owners/remote teams]
Monetization angle: [affiliate/ads/lead-magnet]

Requirements:
- H1 must contain the exact target keyword
- Include 5-8 H2 sections
- Each H2 should include a secondary keyword where natural
- Include data points, statistics, or concrete examples
- End with a clear CTA section
- Write in second person ("you") for engagement
```

**Title Optimization Rules**:
- Maximum 60 characters (Google truncates beyond this)
- Must contain the primary keyword, ideally at the beginning
- Include a power word (Best, Ultimate, Complete, Essential, Top)
- Include the current year if applicable (e.g., "in 2026")
- Examples:
  - Good: `"Best AI Meeting Assistants in 2026 — Complete Comparison"`
  - Bad: `"A Comprehensive Look at Various AI-Powered Meeting Assistant Tools Available Today"`

**Meta Description Rules**:
- Maximum 155 characters
- Must contain the primary keyword within the first 80 characters
- End with a CTA phrase (Compare now, Get started, See the top picks)
- Example: `"Compare the 7 best AI meeting assistants for 2026. See features, pricing, and real team reviews. Find your perfect match →"`

**H2/H3 Keyword Distribution Strategy**:
```
H1: [Primary keyword]                          ← Exact match required
  H2: What Is [Primary Keyword]?               ← Definition section
  H2: Top [Number] [Primary Keyword] Compared  ← Main content
    H3: [Tool/Item 1]                           ← Individual entries
    H3: [Tool/Item 2]
  H2: How to Choose [Primary Keyword]           ← Buying guide
  H2: [Secondary Keyword] Tips                  ← Related topic
  H2: [Primary Keyword] vs [Alternative]        ← Comparison angle
  H2: FAQ                                       ← Featured Snippet target
```

**Content Length Standards**:
| Page Type | Target Words | Minimum Words |
|-----------|-------------|---------------|
| Commercial comparison | 2500-3000 | 2000 |
| Template/checklist | 1500-2000 | 1200 |
| Examples/guide | 2000-2500 | 1500 |
| Pillar page | 3000-5000 | 2500 |
| Tutorial | 2000-3000 | 1500 |

### Step 6: Humanize Content

Run the `humanizer` skill to remove AI patterns:

```
/humanizer

[Paste the generated content]

Instructions:
- Vary sentence length (mix short punchy sentences with longer ones)
- Add conversational transitions ("Here's the thing...", "In practice...")
- Include personal opinion markers ("In my experience...", "What I've found...")
- Replace generic phrases with specific examples
- Ensure Flesch-Kincaid readability score targets 60-70
```

### Step 7: Write FAQ Section

Create at least 5 FAQ entries targeting Featured Snippets:

```typescript
faq: [
  {
    question: 'What is the best [keyword] in 2026?',
    answer: 'The best [keyword] in 2026 is [Tool X] for [use case], [Tool Y] for [use case], and [Tool Z] for [use case]. Each excels in different areas: [Tool X] offers [specific feature], while [Tool Y] provides [specific benefit].'
  },
  {
    question: 'How much does [keyword] cost?',
    answer: 'Most [keyword] tools offer free tiers with limited features. Paid plans typically range from $10-30/month per user. Enterprise plans start at $50/user/month. [Tool X] offers the best free tier, while [Tool Y] provides the best value at $15/month.'
  },
  {
    question: 'Is [keyword] worth it for small teams?',
    answer: '[Specific answer with data points and recommendation]'
  },
  {
    question: '[Keyword] vs [alternative] — which is better?',
    answer: '[Direct comparison with specific criteria]'
  },
  {
    question: 'How do I get started with [keyword]?',
    answer: '[Step-by-step getting started guide in 3-5 sentences]'
  },
],
```

**FAQ Rules**:
- Question must be phrased as a natural search query
- Answer must be 40-60 words (Google Featured Snippet sweet spot)
- First sentence must directly answer the question
- Include at least one number or data point per answer
- Schema is automatically generated by `getFaqSchema()` in site.ts

### Step 8: Set Related Slugs for Cross-Linking

```typescript
relatedSlugs: [
  'related-page-slug-1',   // Must exist in pages[] or aiToolPages[]
  'related-page-slug-2',
  'related-page-slug-3',
  'related-page-slug-4',   // Target 3-5 related slugs per page
],
```

**Cross-linking rules**:
- Minimum 3 related pages per page
- Maximum 5 related pages per page
- At least 1 must be a commercial/affiliate page
- At least 1 must be an informational/template page
- Bidirectional: if page A links to page B, page B should link back to page A

### Step 9: Validate Data Integrity

```bash
npm run validate
```

This runs `scripts/validate-pages.ts`. It checks:
- All required fields are present and non-empty
- All `relatedSlugs` reference existing pages
- No duplicate slugs
- Proper `PageCategory` and `MonetizationPrimary` values
- FAQ entries have both question and answer

**Fix any errors before proceeding.**

### Step 10: Build the Site

```bash
npm run build
```

Expected output:
```
 astro build
✓ Built in X.XXs
  dist/
  ├── pages/your-new-slug/index.html
  ├── sitemap.xml
  └── robots.txt
```

**Build must complete with 0 errors.** If TypeScript errors occur, see [Chapter 9: Troubleshooting](#chapter-9-troubleshooting).

### Step 11: Deploy to Cloudflare Pages

```bash
git add -A
git commit -m "feat: add [slug] page — [target keyword] ([category])"
git push origin main
```

Cloudflare Pages auto-deploys from `main` branch. Deployment typically completes within 1-3 minutes.

**Commit message convention**:
```
feat: add [slug] page — [description]
fix: correct [slug] page data — [what was wrong]
content: update [slug] — [what changed]
chore: update sitemap/schema/styles
```

### Step 12: Post-Deploy Verification

Run these checks after deployment:

1. **Page accessible**: `curl -I https://signalforges.com/pages/[slug]/` → expect `200 OK`
2. **Sitemap updated**: `curl https://signalforges.com/sitemap.xml` → verify new URL present
3. **Schema valid**: Visit `https://search.google.com/test/rich-results?url=https://signalforges.com/pages/[slug]/`
4. **Mobile friendly**: Visit `https://search.google.com/test/mobile-friendly?url=https://signalforges.com/pages/[slug]/`
5. **Page speed**: Visit `https://pagespeed.web.dev/analysis?url=https://signalforges.com/pages/[slug]/`

## Chapter 3: Content Source 2 — AI Tools Comparison Pipeline

### Overview

Produce comparison, review, and pillar content across 5 topic clusters for AI tools. These pages carry high commercial intent and strong affiliate potential.

### Topic Cluster Map

| Cluster | Target Keywords | Content Types | Revenue Potential |
|---------|----------------|---------------|-------------------|
| **Chatbot** | chatgpt vs claude, best ai chatbot, gemini vs chatgpt | comparison, review, pillar | High (affiliate + ads) |
| **Coding** | best ai coding tools, github copilot vs cursor, ai code assistant | comparison, review, tutorial | Very High (dev tools CPC $5-15) |
| **Writing** | best ai writing tools, jasper vs copy-ai, grammarly alternatives | comparison, review, pillar | High (affiliate 30-50%) |
| **Image** | best ai image generators, midjourney vs dalle, ai art tools | comparison, review, gallery | Medium (ads + affiliate) |
| **Productivity** | best ai productivity tools, notion ai vs clickup, ai meeting tools | comparison, review, pillar | High (B2B affiliate) |

### Step 1: Select Comparison Topic

Choose a topic from the cluster map above. Priority:
1. Highest search volume keywords first
2. Keywords with CPC > $5 preferred
3. Ensure cluster balance — don't over-concentrate on one cluster

### Step 2: Determine Content Type

| Content Type | `category` | `aiToolMeta.type` | Use When |

| Head-to-head comparison | `'ai-comparison'` | `'comparison'` | Comparing 2-4 specific tools |
| Pillar/roundup | `'ai-pillar'` | `'pillar'` | "Best X tools" listicle (5-10 tools) |
| Single tool review | `'ai-review'` | `'review'` | Deep-dive on one tool |

Monitor GitHub trending repositories → produce technical content (analysis, comparisons, tutorials, roundups) → capture developer search traffic → monetize via ads and tool affiliates.

### Step 2: Filter — Which Repos to Cover

**Hard Filters** (must pass ALL):

| Criterion | Threshold | Rationale |
|-----------|-----------|-----------|
| Total stars | > 500 | Minimum community validation |
| Today's new stars | > 50 | Confirms active trending momentum |
| Relevance | AI, developer tools, or productivity related | Must fit site niche |
| License | Open source (MIT, Apache, GPL) | Ensures we can discuss freely |

**Scoring for Priority** (after passing hard filters):

| Factor | Weight | Scoring |
|--------|--------|---------|
| Today's stars | 35% | 50-100 = 2pt, 100-500 = 3pt, 500-1K = 4pt, >1K = 5pt |
| Total stars | 20% | 500-5K = 2pt, 5K-20K = 3pt, 20K-50K = 4pt, >50K = 5pt |
| Niche relevance | 25% | Tangential = 2pt, Related = 3pt, Core AI/dev tool = 5pt |
| Content potential | 20% | Low = 1pt, Medium = 3pt, High = 5pt |

**Priority Actions**:
- Score ≥ 3.5 → Write immediately (deep analysis or tutorial)
- Score 2.5-3.4 → Queue for weekly roundup
- Score < 2.5 → Skip

## Chapter 6: Content Quality Checklist

### 6.1 SEO Elements Checklist

| Element | Requirement | Validation Method |
|---------|-------------|-------------------|
| `<title>` tag | ≤ 60 chars, contains primary keyword, includes year if relevant | `page.title.length <= 60` |
| Meta description | ≤ 155 chars, keyword in first 80 chars, ends with CTA | `page.description.length <= 155` |
| H1 | Exactly 1 per page, contains primary keyword | Rendered from `page.title` |
| H2 tags | 5-8 per page, each contains a keyword variant | Count `sections.length` |
| H3 tags | Used for sub-items within sections | Check section content structure |
| Keyword density | 1-2% of total word count | Manual count or tool |
| Canonical URL | Present and correct | Auto-set by `buildCanonicalUrl()` |
| Schema markup | Article + FAQ minimum, Review + Product for tool pages | Auto-generated by schema functions |
| Sitemap | Page URL present in sitemap.xml | Auto-generated by `sitemap.xml.ts` |
| Robots.txt | Not blocking any content pages | Auto-generated, allows all |
| Open Graph | og:title, og:description, og:type, og:url present | Set in `BaseLayout.astro` |
| Twitter Card | summary_large_image | Set in `BaseLayout.astro` |

### 6.2 Readability Checklist

| Metric | Target | Notes |
|--------|--------|-------|
| Flesch-Kincaid Grade Level | 8-10 (8th-10th grade) | Accessible but not dumbed down |
| Flesch Reading Ease | 60-70 | "Standard" to "Fairly Easy" |
| Average sentence length | 15-20 words | Mix short (5-10) with longer (25-30) |
| Paragraph length | 2-4 sentences max | Wall of text kills engagement |
| Bullet/list usage | At least 1 list per 500 words | Scannable content wins |
| Subheading frequency | Every 200-300 words | Keeps readers oriented |
| Active voice | > 80% of sentences | Passive voice weakens writing |

### 6.3 CTA Effectiveness Checklist

| Check | Requirement |
|-------|-------------|
| CTA count | At least 2 per page (hero CTA + in-content CTA) |
| CTA placement | One above fold, one after main comparison/review |
| CTA text | Action-oriented verb + benefit (e.g., "Try Free — No Credit Card") |
| Affiliate links | `rel="nofollow sponsored"` on all |
| UTM tracking | `?utm_source=signalforges&utm_medium=[page-type]&utm_campaign=[slug]` |
| Link testing | All CTAs resolve to correct destination (no 404s) |

### 6.4 Schema Completeness Checklist

| Page Type | Required Schema | Function |
|-----------|----------------|----------|
| All pages | `Article` | `getArticleSchema(page)` |
| All pages | `FAQPage` | `getFaqSchema(page)` |
| AI comparison/review | `ItemList` of `Review` | `getReviewSchema(page)` |
| Tool card pages | `SoftwareApplication` | `getProductSchema(tool)` |
| All pages | `BreadcrumbList` | `getBreadcrumbSchema(page, siteUrl)` |

**How to verify**: Paste page URL into https://search.google.com/test/rich-results

### 6.5 Mobile Friendliness Checklist

| Check | Requirement |
|-------|-------------|
| Viewport meta | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` ✓ (in BaseLayout) |
| Responsive tables | Tables wrapped in `.table-wrap` with horizontal scroll |
| Touch targets | Buttons/links at least 44×44px |
| Font size | Body text ≥ 16px on mobile |

## Chapter 9: Troubleshooting

### 9.1 Common Build Errors

**Error: TypeScript type mismatch in SeoPage**
```
error TS2322: Type '"new-category"' is not assignable to type 'PageCategory'
```
**Fix**: Ensure `category` uses one of the defined values:
```typescript
type PageCategory = 'commercial' | 'template' | 'examples' | 'comparison' | 'checklist'
  | 'ai-comparison' | 'ai-review' | 'ai-pillar' | 'github-trending' | 'tutorial';
```

**Error: Missing required field in SeoPage**
```
error TS2741: Property 'targetKeyword' is missing in type
```
**Fix**: Check the SeoPage type definition and ensure all required fields are present. Use the complete template from Chapter 3, Step 3.

**Error: Import not found**
```
error TS2305: Module '"../lib/site"' has no exported member 'xxx'
```
**Fix**: Check `src/lib/site.ts` exports. All available exports:
```typescript
// Types
export type { MonetizationPrimary, PageCategory, HeroMetric, FeatureCard, PageSection, ToolCardData, PricingPlan, UseCaseItem, SeoPage }

// Values
export { siteConfig, pages, pageMap }

// Functions
export { getAllPageEntries, buildCanonicalUrl, getFaqSchema, getArticleSchema, getReviewSchema, getProductSchema, getBreadcrumbSchema }
```

### 9.2 TypeScript Type Errors

**Error: ratings Record type mismatch**
```
Type 'number' is not assignable to type 'Record<string, number | string>'
```
**Fix**: Ensure `ratings` is structured as `Record<string, Record<string, number | string>>`:
```typescript
ratings: {
  'ToolName': {
    'Dimension1': 8,        // number
    'Dimension2': 'N/A',    // or string
  },
}
```

**Error: aiToolMeta type not matching**
```
Type '"new-type"' is not assignable to type '"comparison" | "pillar" | "review" | ...'
```
**Fix**: `aiToolMeta.type` must be one of:
```typescript
type: 'comparison' | 'pillar' | 'review' | 'trending-digest' | 'setup-guide'
```

### 9.3 Page Not Displaying

**Symptom**: New page returns 404 after build.

**Debugging steps**:
1. Verify slug exists in `pages[]` or `aiToolPages[]`:
   ```bash
   grep -n "slug: 'your-slug'" src/lib/site.ts src/lib/ai-tools-data.ts
   ```
2. Verify `getAllPageEntries()` includes the page:
   ```typescript
   // In site.ts, line 2789-2791:
   export function getAllPageEntries(): SeoPage[] {
     return [...pages, ...aiToolPages];
   }
   ```
3. Check if `ai-tools-data.ts` exports correctly:
   ```typescript
   // ai-tools-data.ts must export:
   export const aiToolPages: SeoPage[] = [/* ... */];
   ```
4. Check build output:
   ```bash
   ls dist/pages/your-slug/
   # Should contain index.html
   ```
5. Verify no duplicate slugs:
   ```bash
   grep "slug:" src/lib/site.ts src/lib/ai-tools-data.ts | sort | uniq -d
   ```

### 9.4 Schema Validation Failed

**Symptom**: Google Rich Results Test shows errors.

**Common fixes**:
1. **Missing FAQ answers**: Ensure every FAQ entry has both `question` and `answer` non-empty.
2. **Invalid Review schema**: Ensure `aiToolMeta.comparisonData.ratings` has numeric values (not undefined).
3. **Breadcrumb error**: Ensure `siteConfig.siteUrl` is set correctly (check `.env` or `astro.config.mjs`).

**How to test locally**:
```bash
npm run build
# Then check dist/pages/[slug]/index.html
# Search for: <script type="application/ld+json">
# Paste the JSON into https://search.google.com/test/rich-results
```

### 9.5 Sitemap Not Updated

**Symptom**: New page missing from sitemap.xml.

**Cause**: Sitemap is generated dynamically by `src/pages/sitemap.xml.ts` which calls `getAllPageEntries()`.

**Debugging**:
1. Ensure the new page is in `pages[]` or `aiToolPages[]`.
2. Rebuild: `npm run build`
3. Check: `cat dist/sitemap.xml | grep your-slug`
4. If using `ai-tools-data.ts`, verify it's imported in `site.ts`:
   ```typescript
   // site.ts line 1:
   import { aiToolPages } from './ai-tools-data';
   ```

### 9.6 Cloudflare Pages Deployment Failed

**Common causes and fixes**:

| Error | Cause | Fix |
|-------|-------|-----|
| Build failed | TypeScript errors | Run `npm run build` locally first, fix all errors |
| Build timeout (> 20 min) | Too many pages or heavy processing | Optimize data, reduce build scope |
| Deploy stuck | Cloudflare cache issue | Trigger manual deploy from CF dashboard |
| 404 after deploy | Wrong output directory | Ensure CF Pages build output is set to `dist` |
| Environment variable missing | `.env` not set in CF | Add `PUBLIC_SITE_URL=https://signalforges.com` in CF Pages > Settings > Environment variables |

**Cloudflare Pages Settings Reference**:
```
Build command: npm run build
Build output directory: dist
Node.js version: 18 (or latest LTS)
Environment variables:
  PUBLIC_SITE_URL = https://signalforges.com
  SITE_URL = https://signalforges.com
```
