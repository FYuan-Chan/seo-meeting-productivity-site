## Why It Matters

TypeScript is currently implemented in TypeScript (self-hosted). Porting to Go could mean:

1. **Faster type checking** — Go is faster than TypeScript for computation
2. **Better IDE performance** — Faster language server
3. **Lower memory usage** — Go is more memory-efficient
