## 我实际用了哪些技能？

### 1. to-prd：生成PRD文档 ⭐⭐⭐⭐⭐

**体验**：这是我用得最多的技能。

**怎么用**：
```bash
# 安装
npx skills@latest add mattpocock/skills/to-prd

# 使用
# 在Claude Code中描述你的项目，然后说：
"使用to-prd技能，把我们的讨论转成PRD"
```

**实际效果**：
- 输入：我们正在开发一个AI代码审查工具
- 输出：完整的PRD文档，包括产品概述、功能需求、优先级

**我的评价**：生成质量不错，能用。但还是要根据实际情况调整，不能完全照搬。

### 2. to-issues：拆解成Issues ⭐⭐⭐⭐

**体验**：配合to-prd用，效果更好。

**怎么用**：
```bash
# 安装
npx skills@latest add mattpocock/skills/to-issues

# 使用
"使用to-issues技能，把上面的PRD拆解成GitHub Issues"
```

**实际效果**：
- 输入：PRD文档
- 输出：多个GitHub Issues，包括标题、描述、标签、预估时间

**我的评价**：拆解得挺合理，但标签和预估时间需要手动调整。
